🔹 Node Nedir? (En Basit Haliyle)

Node, bir işlemi temsil eden kutudur. n8n’deki her işlem bir node’dur.

  • “Veri al” → bir node
  • “Mesaj gönder” → bir node
  • “Bir şeyi kontrol edin” → yine bir node

Düşün: Bir işin her adımı bir kutu. Kutuları birbirine bağlayındık mı iş akışı çalışır!

n8n ile Otomasyon 4. Bölüm - Koşullar ve Akıllı Akışlar

Artık otomasyon kurabiliyoruz. Şimdi işleri biraz daha zeki hale getirme zamanı. Her veri aynı değildir, bazen “şu olursa bunu yap, değilse yapma” demeliyiz. İşte şimdi bunu öğreneceğiz.

🎯 Bu Bölümde Ne Öğreneceğiz?

  • Set node ile veri oluşturmayı ve düzenlemeyi
  • IF node ile koşul eklemeyi
  • Switch node ile çoklu yol oluşturmayı

🔹 Set Node – “Benim Verim Bu!”

Set node, sizin elle veri oluşturmanızı sağlar. Test amaçlı veya veri hazırlamak için çok kullanılır.

Örnek Veri

{
  "isim": "Nefise",
  "puan": 85
}

Nasıl Yapılır?

  1. “+” tuşuna tıklayın → Set node ekleyinyin
  2. Field Name: isim, Value: "Nefise"
  3. Field Name: puan, Value: 85
  4. Çalıştırın ve node çıktısını inceleyin

🔸 IF Node – Şartlı Akış Kur

“Eğer puan 90’dan büyükse tebrik et, değilse farklı mesaj gönder.”

Nasıl Kurulur?

  1. “+” tuşuna tıklayın → IF node ekleyinyin
  2. Koşul ekleyin:
  • Field: puan
  • Condition: larger than
  • Value: 90
  1. True ve False çıkışlarına göre farklı node’lar bağla

🔻 Switch Node – Çoklu Seçenekli Kontrol

Birden fazla değere göre ayrım yapar. Örnek: Puan aralığına göre seviyelendirme.

Nasıl Kurulur?

  1. Switch node ekleyinyin
  2. Switch key: puan
  3. Aşağıdaki gibi değerleri sırayla girin:
  • Case 1: <= 50 → "Kaldın"
  • Case 2: <= 75 → "Geçtin"
  • Case 3: <= 100 → "Harika!"

Her çıkışa farklı aksiyonlar bağlanabilir.

🎯 Mini Görev: Otomatik Tebrik Sistemi

Senaryo: Öğrenci formu doldurdu, puan alanına göre ona otomatik mesaj gidiyor.

Kullanılacak Node'lar

  • Set → puanı 92 olarak ayarlayın
  • IF → puan > 90 kontrolü
  • Telegram →
  • True: “Tebrikler! 🎉”
  • False: “Devam edin, daha iyisini yaparsınız!”

🧠 Bu Bölümde Ne Öğrendik?

  • Set ile örnek veri oluşturduk
  • IF ile veri içeriğine göre koşul ekledik
  • Switch ile birden fazla senaryo için dallanma yaptık
  • Otomasyonumuzu daha zeki hale getirdik

🔹 HTTP Request Node Nedir?

İnternetteki herhangi bir API’ye bağlanmanı sağlar.

Örnek senaryolar:

  • Hava durumu bilgisi çek
  • Döviz kuru al
  • Bir web formuna veri gönder

🛠 Örnek: Hava Durumunu Çekmek

Biz şimdi ücretsiz bir hava durumu API’sinden veri çekeceğiz: https://api.weatherapi.com

1. API Key Al

  1. https://www.weatherapi.com/ adresine gidin
  2. Ücretsiz hesap oluştur
  3. API Key’inizi kopyalayın (örnek: abc123xyz456)

2. HTTP Request Node’u Oluştur

  1. “+” tıklayın → HTTP Request seçin
  2. Ayarları yap:
  • Method: GET
  • URL: 
https://api.weatherapi.com/v1/current.json?key=APIKEYINIZ&q=Istanbul

APIKEYINIZ kısmını kendi anahtarınla değiştir!

  1. Request’i test et → sağda JSON olarak veri gelecek.

🧪 Örnek Veri Yanıtı

{
  "location": {
    "name": "Istanbul",
    "country": "Turkey"
  },
  "current": {
    "temp_c": 23.0,
    "condition": {
      "text": "Partly cloudy"
    }
  }
}

🔎 Bu Veriyi Kullanmak

Bir sonraki node’da bu veriyi nasıl kullanacağını öğrenelim. Örneğin Telegram node'una şu mesajı yazın:

İstanbul’da hava durumu: {{$json["current"]["condition"]["text"]}}  
Sıcaklık: {{$json["current"]["temp_c"]}}°C

🔹 Webhook Nedir?

Webhook, dış sistemlerden gelen veriyi anlık olarak yakalayan bir kapıdır.

Webhook node, size özel bir URL oluşturur. Bir sistem bu URL’ye veri gönderirse, n8n otomatik olarak tetiklenir.

🛠 Webhook Kurulumu

1. Webhook Node'u Ekle

  1. “+” tıklayın → Webhook node seçin
  2. Ayarları yap:
  • HTTP Method: POST
  • Path: ornek-form (bu URL’nin son kısmı olur)

2. Test Moduna Al

Node’un sağ üstünden Execute Node’a tıklayın Bu, gelen veriyi dinlemeye başlar

3. Webhook URL’yi Kopyala

Sağ üstte Test URL gözükecek Örn:

https://n8n.app.n8n.cloud/webhook-test/ornek-form

🧪 Basit HTML Form ile Test

Aşağıdaki kodu kendi sitene ya da CodePen’e yapıştırın:

<form action="https://n8n.app.n8n.cloud/webhook-test/ornek-form" method="post">
  <input name="isim" placeholder="Adınız">
  <input name="email" placeholder="E-posta">
  <button type="submit">Gönder</button>
</form>

Formu doldur → Gönder → n8n'de veri yakalanır ✅

🔁 Otomasyonu Tamamla

  • Webhook → E-posta (form içeriğini gönder)

🔧 API Nedir?

API (Application Programming Interface), iki yazılımın birbiriyle iletişim kurmasını sağlayan bir "aracı"dır.

🍕 Günlük Hayattan Örnek:

  • Siz restorana gittiniz.
  • Garsona "bir pizza" dediniz.
  • Garson mutfağa gidip pizzanızı size getirdi.

Burada:

  • siz = Kullanıcı
  • Garson = API
  • Mutfak = Sistem

Garson (API) olmasaydı, Siz mutfağa girip işi kendiniz yapacaktınız. İşte API’ler, sistemler arasında bu işleri halleden "garsonlardır".

🤖 n8n API’leri Nasıl Kullanır?

n8n, hazır "garsonlar" (node’lar) sunar. Bu node’lar, API'lerle konuşur.

🔁 Örnek Senaryo:

“Stripe’tan ödeme gelince, Philips Hue ışığını yak.”

  • Stripe Trigger node'u → API ile Stripe'tan gelen ödemeyi algılar
  • Philips Hue node'u → API ile ışık sisteminize komut gönderir

Bu işlemin hiçbir satır kod yazmadan gerçekleşmesini sağlar.

Auth

BONUS 1 - n8n İle Otomasyon - Authentication

1. Authentication nedir?

“Authentication” demek, kimliğinizi kanıtlamak demek. Tıpkı bir kapıyı açmak için anahtar kullanmaya benzer. Siz kim olduğunuzu kanıtlarsınız, sistem de size kapıyı açar.

  • Eğer içerik herkese açık (mesela google.com anasayfası), anahtar gerekmez.
  • Ama özel bilgi istiyorsanız (mesela sizin Gmail hesabınıza bakmak), sistem der ki: “Önce bana kim olduğunuzu kanıtlayın.”

2. n8n’de bu iş nasıl oluyor?

n8n bir yardımcı robot gibi çalışıyor. Siz ona “Bu servisten veri al” diyorsunuz. Robot da soruyor: “Peki, bu servisin kapısını açacak anahtarı bana verecek misiniz?”

İşte bu anahtar credentials oluyor. Yani kullanıcı adı–şifre, token ya da izin bağlantısı.

3. Kullanabileceğiniz Anahtar Çeşitleri

Kapı açmanın birkaç yolu var:

  1. Basic Auth (En basit yöntem)
  • Kullanıcı adı + şifre yazarsınız.
  • Zayıf güvenlikli, ama eski sistemlerde işe yarar.
  1. Bearer Token (Tek anahtar)
  • Servis size bir sihirli anahtar verir (token).
  • n8n bu anahtarı her isteğe ekler.
  • En kolay kullanılan yöntemdir.
  1. Header Auth (Özel başlık)
  • Bazı servisler anahtarı özel bir isimle ister (ör. X-API-Key).
  • n8n buna izin verir.
  1. Query Auth (Linkin içine gömülen anahtar)
  • Anahtarı linkin sonuna ekler (?apikey=12345).
  • Pek güvenli değil çünkü linki gören herkes anahtarı görür.
  1. OAuth2 (En modern ve güvenli)
  • Siz n8n’e “Benim adıma Google’a bağlanabilirsin” dersiniz.
  • Google sizi kendi ekranına yönlendirir, “Bu robota şu izinleri veriyor musunuz?” diye sorar.
  • Siz onaylarsınız, Google n8n’e özel bir anahtar verir.
  • Bu anahtar sayesinde n8n sizin yerinize işlem yapar.
  • En büyük servisler (Google, Slack, Microsoft vb.) bu sistemi kullanır.

4. Hangi Anahtarı Seçmeli?

  • Servis için hazır bir ayar varsa (mesela Airtable, Notion, Slack), onu seçin → işin çok kolaylaşır.
  • Eğer yoksa:
  • Modern servisler → OAuth2
  • Basit API’ler → Bearer Token
  • Çok eski sistemler → Basic Auth veya Digest

5. Hangi Yöntemi Kullanacağımızı Nasıl Bileceğiz?

Burada sihirli cevap: Servisin dökümantasyonu. Her servis kendi kapısını nasıl açacağını yazar. Siz de n8n’de buna uygun anahtarı seçersiniz.

  • Dokümanda “Bearer Token” veya “API Key” yazıyorsa → _Bearer Auth_ ya da _Header Auth_.
  • Dokümanda “API Key in URL” yazıyorsa → _Query Auth_.
  • Dokümanda “OAuth2” diyorsa → _OAuth2_.
  • Dokümanda “Basic Authentication (username & password)” diyorsa → _Basic Auth_.

👉 Eğer servis için n8n’de zaten hazır node varsa, doğru yöntem sizin yerinize otomatik seçilmiş olur. Sadece gerekli bilgiyi girmeniz yeterlidir.

Kısacası:

n8n’de authentication, robotunuz sizin yerinize bir kapıdan geçebilmesi için ona doğru anahtarı vermek gibidir. Bazen bu anahtar bir şifre olur, bazen tek kullanımlık bir kart (token), bazen de büyük bir şirketten alınmış resmi izin belgesi (OAuth2).

İlk bakışta adımlar fazla görünebilir ama bir kez ayarladıktan sonra tekrar tekrar kullanabilirsiniz.

Adım 1 – Proje Oluştur

  • cloud.google.com’a gidin ve hesabınıza giriş yapın.
  • Sol üstteki Project menüsünden → New Project seçin → isim verin ve Create’e tıklayın.
  • Birkaç saniye içinde proje hazır olacak.

Adım 2 – Gerekli API’leri Etkinleştir

  • Menüden APIs & Services → Enable APIs and Services yolunu izleyin.
  • Buradan kullanacağınız servisleri arayın ve enable yapın:
  • Google Sheets API
  • Google Drive API
  • (İhtiyacınıza göre Calendar veya Gmail de açabilirsiniz.)

Adım 3 – OAuth Consent Screen Hazırla

Şimdiye kadar n8n’de bir credential oluşturduk ve içine Google’dan aldığımız bilgileri koyduk. Ama Google hâlâ “Bu uygulama kimdir?” diye soruyor. İşte burada devreye OAuth giriyor.

  • Google’a dönün → OAuth ayarlarını tamamlayın.
  • Bu ayar sayesinde Google, n8n’i tanıyor ve güveniyor.
  • Böylece n8n sizin yerinize Google servislerine bağlanabiliyor.
  • APIs & Services → OAuth consent screen sekmesine girin.
  • Get Started de, uygulamanıza bir isim verin (ör. _n8n App_).
  • Gmail hesabı kullanıyorsanız External, Workspace kullanıyorsanız Internal seçin.
  • Destek e-postanızı girin ve kaydedin.

Consent screen ayarlarını tamamladık. Bu ekran sayesinde Google, uygulamanızı gördü ve kullanıcıya hangi izinleri isteyeceğini biliyor. Şimdi sıra Google’ın n8n’i gerçekten tanıyacağı kimlik kartını oluşturmaya geldi: OAuth Client ID.

Adım 4 – OAuth Client ID Oluştur

Google, n8n’i tanımadığı için bir kimlik isteyecektir. Siz Client ID oluşturunca Google n8n’e bir kimlik kartı verir ve n8n bu kartla sizin adınıza işlem yapabilir. Yani OAuth Client ID, Google’a “Bu uygulama benim” demenin yolu. BU ADIM ÇOK ÖNEMLİ!

  • Credentials → Create Credential → OAuth Client ID yolunu izleyin.
  • Uygulama türü olarak Web Application seçin.
  • Bir isim verin (ör. _n8n Client_).
  • Redirect URI eklemen gerekiyor:
  • Bunun için n8n'e gidin ve yeni bir credential oluştur.
  • Bu adımda hangi uygulama için credential oluşturacaksanız onu seçin. Unutmayın her uygulama için ayrı seçim yapmanız gerekiyor.

-

  • Orada çıkan Redirect URL’yi kopyalayın, Google Console’daki alana yapıştırın.
  • Create de → size Client ID ve Client Secret verecek. bunu kopyalayın.

Adım 5 – Bilgileri n8n’e Gir

  • n8n’de Google Credential ekranına dönün.
  • Client ID değerini yapıştırın.
  • Client Secret değerini almak için console'da clients sayfasına geri dönün ve secret key'i kopyalayın.
  • - n8n’de Google Credential ekranına dönün.
  • Client Secret değerini yapıştırın.
  • Kaydet.

Not:

- OAuth yöntemi, Google’ın belirlediği kural kitabıdır.

- Credential, bu kurala göre hazırlanmış anahtar kutusudur.

Yani ikisi ayrı şeydir ama birbirini tamamlar.

Adım 6 – Google ile Bağlan

  • Credential ekranında Sign in with Google butonuna basın.
  • Google hesabınızla giriş yapın → izinleri seçin.
  • İlk girişte “application blocked” hatası alabilirsiniz (çünkü uygulama test modunda).

-

  • Çözüm: Google Console’da uygulamanızı Publish App yap.

Adım 7 – Bağlantıyı Test Et

  • Tekrar giriş yaptığınızda bağlantı başarılı olacak. Bazen biraz beklemeniz gerekebilir. Endişe etmeyin. Genellikle 10 dakika içinde bağlantı kurabiliyorsunuz.
  • Artık n8n’den Google Sheets oluşturabilir, Drive’daki dosyaları yönetebilir ya da Gmail üzerinden otomatik e-posta gönderebilirsin. Unutmayın her servis için yeni bir credential oluşturman gerekiyor. Google Sheets için oluşturduğunuz credential, Calendar uygulaması için çalışmaz :)

2. OpenAI ChatGPT

n8n’de ChatGPT’yi kullanabilmek için OpenAI API anahtarını (API Key) almak ve bunu n8n’e credential olarak eklemek gerekiyor. Böylece n8n, ChatGPT ile konuşabilir ve sizin workflow’larınızın içinde metin üretebilir.

Not: API Key, ChatGPT’ye erişmek için size verilen gizli anahtardır. Bu anahtar olmadan n8n ChatGPT’ye bağlanamaz.

Adım 1 – Yeni Credential Oluştur

  • n8n anasayfasında Credential bölümüne gel.

-

  • Create New Credential seçeneğine tıklayın.
  • Burada senden OpenAI API anahtarı istenecek.

Adım 2 – OpenAI API Key Al

  • platform.openai.com adresine gidin. (Uygulamanın sayfası değil, işlerin arkaplanı burası.)
  • Hesabınıza giriş yapın (yoksa ücretsiz bir hesap aç).
  • Sağ üstteki profil simgesine tıklayın → View API keys menüsüne girin.
  • Eğer daha önce oluşturmadıysanız Create new secret key seçin.
  • Anahtara bir isim verin (ör. _n8n ChatGPT Key_).

-

  • Create secret key dedikten sonra çıkan uzun anahtarı kopyalayın. ⚠️ Bu anahtar yalnızca bir kez gösterilir. Ve kimse ile paylaşmaman gerekir. Aksi takdirde bir gün uyandığında tüm paranın başkaları tarafından harcandığını görebilirsiniz.

Adım 3 – API Key’i n8n’e Yapıştır

  • Tekrar n8n’e dönün.
  • Credential ekranındaki ilgili alana API key’i yapıştırın.
  • Save de.
  • Bağlantı başarılıysa yeşil bir onay kutusu görürsün.

Adım 4 – Bütçe

  • Eğer “insufficient quota” hatası alırsan bu, hesabında kredi olmadığını gösterir.
  • OpenAI hesabında Billing bölümüne gidin → kredi ekleyin.
  • Tekrar denediğinde başarılı şekilde yanıt alacaksınız.
  • 10$’lık kredi, küçük projeler için uzun süre yeterlidir.
  • Fiyatlandırmayı görmek için OpenAI Pricing sayfasını kontrol edin.

Artık ChatGPT’yi n8n workflow’una bağladınız ✅ -> Bundan sonra ChatGPT, n8n’in içinde sizin robot yardımcınız gibi çalışacak. Ona sorular sorabilir, metin yazdırabilir veya otomatik yanıtlar oluşturabilirsiniz.

3. KIE AI: Burada Erişebilirsiniz

KIE AI, tek bir API anahtarıyla onlarca farklı görsel, video ve müzik modeli kullanmanı sağlar. Normalde her model için ayrı ayrı API key almak gerekirken, KIE AI sayesinde hepsine tek noktadan erişebilirsiniz. Üstelik çok uygun fiyatlı olduğu için içerik üretiminde hızlı ve ucuz denemeler yapabilmek mümkün.

Adım 1 – Hesap Oluştur

  • kie.ai adresine gidin.
  • Google hesabınızla veya e-posta ile ücretsiz kayıt olun.
  • İlk kayıt olunduğunda küçük bir deneme kredisi verilir, hemen test edebilirsiniz.

Adım 2 – API Key Al

  • KIE AI panelinde API Keys bölümüne girin.
  • Create API Key seçeneğine tıklayın.
  • API key için bir isim verin (ör. _n8n KIE Key_).

⚠️ Bu anahtar yalnızca bir kez gösterilir, mutlaka güvenli bir yere kaydedin.

  • Create de → uzun bir anahtar çıkacak.

Adım 3 – n8n’de Credential Oluştur

  • n8n’de Credentials → Create New Credential yolunu izleyin.
  • Servis olarak Header Auth (manual bağlantı için) seçin.

👉 Örnek: Bearer e9ehh93745ddgefddfk

  • Name bölümüne "Authorization" yazın.
  • Value alanına Bearer yazın, yanına bir boşluk bırak ve hemen ardından kendi API Key’inizi ekleyin.
  • Kaydet.

Artık KIE AI’yı n8n’e bağladınız ✅ Bundan sonra tek bir API key ile:

  • Görseller oluşturabilir,
  • Videolar üretebilir veya düzenleyebilir,
  • Gerekirse müzik modellerini bile kullanabilirsiniz.

KIE AI’nin avantajı, tüm modelleri tek anahtardan yönetebilmen. Yani MidJourney, Runway, VO3 gibi farklı servisler için ayrı ayrı hesap açmana gerek yok.

4. Fal AI

Fal AI, 600’den fazla görsel, video, ses ve müzik modeline tek bir API ile erişmeni sağlar. Özellikle görselleri videoya dönüştürmek için kullanılır.

n8n’de kullanabilmek için bir API Key oluşturup bunu credential olarak eklemen gerekir.

Adım 1 – API Key Oluştur

  • fal.ai hesabınıza giriş yapın.
  • Sağ üstteki API Keys bölümüne gidin.
  • Create API Key butonuna basın.
  • Anahtarınızı kopyalayın → güvenli bir yere kaydedin. (size yalnızca bir kez gösterilir.)

Adım 2 – n8n’de Credential Oluştur

  • n8n’de Credentials → Create New Credential yolunu izleyin.
  • Generic Credential Type seçin.
  • Açılan listeden Header Auth seçin.
  • + Create new credential de.

Şu değerleri girin:

  • Name: Authorization

👉 Örnek: Key 9dj32f8sd9f23jsdf...

  • Value: Key yazın, ardından bir boşluk bırak ve kendi API Key’inizi ekleyin.
  • Save diyerek kaydedin.
  • Artık n8n, Fal AI ile bağlanmaya hazır.

🎁 Bonus: Auth Değerleri Neden Farklı?

API anahtarını n8n’de girerken bazen Bearer, bazen Key, bazen de hiçbir şey yazmıyoruz. Peki neden?

Her servis, güvenlik sisteminizi biraz farklı tasarlıyor:

  • Bearer → Servis “bu bir kimlik kartıdır” demek için başına _Bearer_ koymamızı ister.
  • Key → Bazı servisler “anahtar” kelimesini önde görmek ister.
  • Hiçbir şey → Bazıları ise doğrudan anahtarı kabul eder, ekstra ibare istemez.

👉 Yani burada bir yanlışlık yok. Sadece her API’nin “şifreyi sunma şekli” farklı. Biz de n8n’de, servisin istediği formata göre Value kısmını dolduruyoruz.

Peki biz hangisini yazacağımızı nasıl bileceğiz? Bunun cevabı için her zaman ilgili servis sağlayıcının resmi dökümantasyonuna bakmamız gerekir. Orada hangi formatı istedikleri net şekilde yazar.
